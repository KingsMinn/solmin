# TLS
>Transport Layer Security
>전송 계층 보안

- **약자 풀이:** `Transport Layer`(전송 계층, 즉 4층) + `Security`(보안). 즉, **"4층 배송부에서 보내는 모든 소포를 안전하게 암호화해서 포장하는 기술"** 입니다.
- **성격:** '비밀 문서 전문' 보안 포장팀장.
- **비유:** '비밀 암호 봉투' 또는 '내용물이 보이지 않는 강철 컨테이너'.
- **업무 방식 (TMI):**
    
    1. **역사:** TLS의 원래 이름은 **SSL(Secure Sockets Layer)** 이었습니다. 넷스케이프라는 전설적인 회사가 만들었죠. 이후 SSL의 약점을 보완하고 표준화하여 새로 나온 것이 TLS입니다. 즉, SSL은 구형, TLS는 신형 버전이라고 생각하시면 됩니다. 우리가 쓰는 `HTTPS`의 `S`가 바로 이 SSL/TLS를 의미합니다.
    2. **작동 위치:** TLS는 7층(HTTP)과 4층([[250917-TCP|TCP]]) **사이**에 끼어서 일합니다.
    3. **작동 순서:**
        
        - `TCP`가 먼저 `3-Way Handshake`로 안정적인 통신 라인을 구축합니다.
        - 그 라인 위에서, `TLS`가 자신만의 `TLS Handshake`를 시작합니다.
        - 이 `TLS Handshake` 과정에서 우리가 배웠던 **'[[HTTPS 인증서]]'** 와 **'[[비대칭키]]'** 암호화 기술을 사용해서 서버의 신원을 확인하고, 앞으로 둘만 사용할 **'임시 대칭키(세션 키)'** 를 안전하게 교환합니다.
        - `TLS Handshake`가 끝나면, 이제 7층에서 내려오는 모든 편지 내용물(`HTTP` 요청)은 이 '임시 대칭키'로 암호화 포장된 뒤, 4층의 `TCP` 택배 상자에 담겨 배송되는 것입니다.
