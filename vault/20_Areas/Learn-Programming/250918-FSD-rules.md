# [[250918-FSD|FSD]]의 황금률: "강물은 아래로만 흐른다"
FSD의 가장 중요하고 엄격한 규칙은 바로 **'[[의존성]] 규칙'** 입니다.

> **"모든 계층은 자신보다 '아래'에 있는 계층의 [[모듈]]만 사용할 수 있다."**

- **비유:** 강물은 산(shared)에서 시작해서 바다(app)로, 오직 아래로만 흐릅니다. 5층의 `widgets`는 4층의 `features`나 3층의 `entities`를 가져다 쓸 수 있지만, **절대 6층의 `pages`가 무엇인지 알면 안됩니다.**
- **왜?** 이 규칙은 코드의 재사용성을 극대화하고, 스파게티처럼 얽히는 의존성을 원천 차단합니다. `PostCard` 위젯은 어느 페이지에 갖다 놔도 똑같이 작동해야지, 특정 페이지에 종속되어서는 안 되니까요.
    
또한, 각 '슬라이스'(예: `features/LikePost`)는 `index.ts` 같은 하나의 '공개 창구(Public API)'를 통해서만 자신의 기능을 외부에 노출해야 합니다. 다른 모듈이 슬라이스 내부의 복잡한 파일들을 마음대로 `import` 해가는 것을 막아, 내부 구조를 언제든 안전하게 [[리팩토링]]할 수 있게 해줍니다.
